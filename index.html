Implementación de ORM y Gestión de Datos

Una mirada profunda a la arquitectura de software moderna utilizando Entity Framework y metodologías iterativas.

Grupo Continental Docente: Hugo Espetia

01 Introducción

El proyecto se centra en la optimización del acceso a datos mediante el uso de **Mapeo Objeto-Relacional (ORM)**. Al separar las responsabilidades del sistema en capas, se logra una arquitectura más limpia, escalable y fácil de mantener.

Simplifica la interacción con la BD mediante el mapeo de tablas a objetos de C#.

SQL Server

Motor de base de datos robusto para el almacenamiento y gestión de la persistencia.

Conclusiones

Simplificación

Entity Framework facilitó la interacción mediante LINQ, reduciendo la necesidad de SQL crudo.

Organización

La arquitectura en capas permitió una separación clara entre la lógica y los datos.

Desafíos

Las relaciones complejas y claves foráneas requirieron ajustes manuales en el Fluent API.

Control

El uso disciplinado de Git garantizó un avance organizado y sin pérdida de información.
